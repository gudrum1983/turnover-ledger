# Vue: Стилизация через пропсы — краткая памятка

Цель: быстро вспомнить основные способы менять стили компонента через пропсы, с примером из этого репозитория.

## 1) Динамические классы (`:class`)
- Подходит, когда проп отвечает за конечный набор вариантов (variants).
- Удобно сочетается с BEM-неймингом.

Пример (BaseDivider):
- `:class="[\`BaseDivider_style_${lineStyle}\`, { BaseDivider_vertical: isVertical }]"`
- Файл: `src/shared/ui/display/BaseDivider.vue`

## 2) Inline стили / CSS-переменные (`:style` + custom properties)
- Для динамических значений, которые нужно использовать внутри CSS-правил.
- Логика остается в script, раскладка — в стилях.

Пример (BaseDivider):
- `:style="cssVars"`, где `cssVars` задает `--divider-color`
- Файл: `src/shared/ui/display/BaseDivider.vue`

## 3) `v-bind()` внутри `<style>`
- Для значений, вычисленных в script и напрямую используемых в CSS.

Пример (BaseDivider):
- `--divider-thickness: v-bind(clampedThicknessPx);`
- Файл: `src/shared/ui/display/BaseDivider.vue`

## 4) Проброс `class` / `style` от родителя (`$attrs`)
- Полезно, когда компонент-обертка должен принимать внешнюю стилизацию.
- Используйте `inheritAttrs` или явный `v-bind="$attrs"`.

## 5) CSS Modules (`<style module>`)
- Скоупленные классы с автоматической генерацией имен и доступом через `$style`.
- В этом репозитории встречается редко, но полезно для строгой изоляции.

## 6) Темизация через глобальные CSS-переменные
- Общие дизайн-токены (цвета, отступы), заданные на уровне приложения.
- Компоненты читают токены без явных пропсов.

## Когда что выбирать
- `:class` — если есть фиксированный набор вариантов (например, `solid | dashed | dotted`).
- `:style` + CSS-переменные — если нужно динамическое значение, но логика должна остаться в CSS (цвета, толщина, размеры).
- `v-bind()` в `<style>` — если значение вычисляется в script и его удобно сразу прокинуть в CSS как переменную.
- `$attrs` — если компонент должен пропускать внешние `class`/`style` без знания о них.
- CSS Modules — если нужна жесткая изоляция имен классов.
- Глобальные CSS‑переменные — если это дизайн-токены, которые используются многими компонентами.

## Что общее с React, а что уникально для Vue
- Общее с React:
  - Динамические классы.
  - Inline стили.
  - CSS‑переменные.
  - CSS Modules.
  - Проброс `className` / `style` от родителя.
- Уникально для Vue:
  - `v-bind()` внутри `<style>` в SFC (прямая связь script → CSS без ручного инлайна).
  - Механика `scoped`-стилей (в React это не встроено, обычно через CSS Modules или CSS-in-JS).
