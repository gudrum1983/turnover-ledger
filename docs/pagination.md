Для твоего требования “как у формы” + **автоперенос страниц** + **промежуточные итоги** + **портрет/альбом** + **строки разной высоты** — чисто CSS-печатью (HTML → `window.print`) это довести до “железобетонно как бухгалтерия” почти невозможно, потому что браузер **не даёт надёжно узнать, где именно случился разрыв страницы** и **вставить строку “Промежуточный итог” ровно перед разрывом**.

Но можно сделать архитектуру, которая в реальности работает:

## Самый надёжный путь: “пагинация в JS” + печать готовых страниц

Ты **сама режешь** строки на страницы (с учётом реальной высоты), вставляешь:

* “Итого на предыдущей странице”
* “Перенос итога”
* повторяющуюся шапку

А потом печатаешь уже не таблицу, а **набор страниц** (div’ы A4), где всё заранее расставлено.

### Почему это работает

* ты измеряешь DOM-элементы (`getBoundingClientRect()`), значит знаешь реальную высоту строки с переносами
* можно поддержать **portrait/landscape** просто разными лимитами по высоте/ширине
* промежуточные итоги считаются на уровне данных и вставляются как отдельные “служебные строки”

## Как бы я это спроектировала в твоём Vue+FSD

### 1) Модель для печати

Вместо “rows[]” напрямую печатать, ты строишь:

```ts
type PrintLine =
  | { kind: 'row'; rowId: string }
  | { kind: 'subtotal'; label: string; sums: {...} }
  | { kind: 'carry'; label: string; sums: {...} } // перенос с предыдущей
```

А результат пагинации:

```ts
type PrintPage = {
  pageNo: number
  orientation: 'portrait' | 'landscape'
  carryFromPrev?: {...}
  lines: PrintLine[]
  footerSums?: {...}
}
```

### 2) Алгоритм “разложить по страницам”

Идея: сначала рендеришь строки в “скрытом измерительном контейнере”, меряешь их высоты, потом режешь.

* рендеришь строку (так же, как в печати) в hidden wrapper (`visibility:hidden; position:absolute; left:-99999px`)
* измеряешь высоту каждой строки (с переносами)
* набираешь строки, пока не превышен лимит высоты страницы
* если не влезает — закрываешь страницу:

    * вставляешь subtotal в конец
    * начинаешь новую страницу:

        * вставляешь carry (перенос) в начало
        * потом продолжаешь строки

**Шапку не меряешь каждый раз**, она фиксированная (просто вычитаешь её высоту из доступного места).

### 3) Печать

В UI печати ты показываешь:

* `<section class="page A4 portrait|landscape">`
* внутри: header + table(lines) + footer
* CSS: `page-break-after: always;` для `.page`

---

## Ориентация и размеры

Ты уже делала динамическую ориентацию через `@page size: A4 landscape/portrait`.
В этом подходе ещё добавляется **разный “лимит по высоте контента”**:

* Portrait: меньше ширина → чаще переносы → строки выше → меньше строк на страницу
* Landscape: больше ширина → меньше переносов → больше строк на страницу

То есть ориентация влияет на *измерение* (и это ок — ты всё равно меряешь реальный DOM).

---

## Практичные правила для строк с переносами

Чтобы переносы были предсказуемыми:

* у описания задай фиксированную ширину колонки
* `white-space: normal; word-break: break-word;`
* для чисел: `white-space: nowrap;` (чтобы суммы не ломались)

---

## А если без JS-пагинации?

Тогда реально остаётся только:

* `thead { display: table-header-group }` чтобы повторять шапку
* `tr { break-inside: avoid }` чтобы не рвать строку посередине
* и смириться, что “перенос итога” будет *не идеальный*, а просто “итоги внизу страницы через CSS” — но это не то, что ты хочешь.
