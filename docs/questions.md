Старт приложения

      1. Стор инициализируется дефолтами.
      2. Пытаемся гидратировать из localStorage (если нет открытого JSON-документа).
      3. Форма читает значения только из стора.

Пользователь редактирует форму

      1. Изменение → сразу в стор.
      2. Watcher (лучше с debounce) пишет стор в localStorage.

Импорт JSON

      1. Парсим и валидируем JSON.
      2. Полностью перезаписываем стор данными из JSON.
      3. Сразу сохраняем в localStorage (чтобы состояние переживало перезагрузку).
      4. Форма автоматически обновится, потому что она связана со стором.

Экспорт JSON

      1. Берём данные из стора.
      2. Формируем JSON и скачиваем.

Логика для запросов по конвертации без кнопки:

- Делать запрос только когда значение реально изменилось.
- Триггерить на blur (потеря фокуса) или с небольшим дебаунсом при вводе.
- Хранить lastConvertedInput (например строку суммы + валюту + дату), чтобы не дёргать API повторно.

Пример стратегии:

- Пользователь вводит сумму.
- На blur:
    - сравниваешь inputValue с lastConvertedValue.
    - если совпадает → запрос не отправляешь.
    - если отличается → отправляешь, сохраняешь lastConvertedValue.

Это не требует TanStack Query. Но если запросов будет много и нужно кэширование по ключам (date+currency+amount), TanStack Query упростит: там есть key‑cache и dedupe.
